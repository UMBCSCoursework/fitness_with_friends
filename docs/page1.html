1. User Analysis

  • Identify the characteristics of your user population using personas. Describe age, gender, education, physical limitations, computer experience, motivation, domain experience, where they will use the software, if there are any factors distracting them, relationships, and communication patterns. Describe one persona per user class.

For the user analysis, our team members interviewed 5 potential users of our "Fitness with Friends" application. Out of the collected information, we identified the following four user classes:

  1. A Mindfulness-Oriented User interested in the Fitness Progress of Friends

  2. Growth and Accessibility minded Professional Instructor

  3. A Fitness Tracking, Competitive User

  4. Seasonal Instructor, Curious About Meeting New People

2. Task Analysis

  • Determine the tasks of the problem you've chosen and identify at least 3 high-level tasks. Make a detailed task analysis for these tasks with textual or diagrammatic HTA (Hierarchical Task Analysis). The selected tasks should not be as simple as login/logout etc. For example, in a recipe site, the most central, interesting tasks might be editing a recipe, finding a recipe, and using a recipe (to actually cook). If you can't find 3 interesting tasks, then your problem may be too small to serve as a good project, and you should rethink it. Every task should have a goal and subtasks.

Here we outline Hierarchical Task Analysis for 3 tasks:

  1. A workout with a random person

  2. Pick a personal trainer

  3. Track and compare your progress

Plan 0 for Task 1: when duration of a workout matters for a user : 1 → 2 → 3 (including subtasks) → 4 → 5 → 6 → 7 → 8

Plan 1 for Task 1: when duration of a workout does not matter for a user : 1 → 2 → 3 (including subtasks)→ 5 → 6 → 7 → 8

Plan 0 for Task 2: check trainers until proper one is found, repeat: 4.1 → 4.2 → 4.3 → 4.4

Plan 0 for Task 3: to track self fitness progress: 1 → 2→ 3 (including subtasks) → 5

Plan 1 for Task 1: to compare self progress with friends : 1 → 2→ 4 (including subtasks)→ 5
